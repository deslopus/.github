# Contributing

Deslopus exists to improve how coding agents handle real software work. Contributions should help agents produce cleaner, safer, more maintainable code.

## Useful Contributions

- Prompts and instruction sets for real coding workflows
- Benchmarks for long, multi-step maintenance tasks
- Review checklists for AI-assisted changes
- Agent setup notes, tool configurations, and failure reports
- Examples that show specific behavior before and after a change
- Documentation that makes practices easier to evaluate or reuse

## Quality Bar

- Keep changes scoped and explain the problem they address.
- Include enough context for someone else to reproduce the result.
- Name the agent, model, tools, environment, and repository shape when relevant.
- Prefer concrete examples over broad claims.
- Include verification steps for code, benchmark, or workflow changes.
- Avoid secrets, credentials, private source code, and proprietary data unless you have explicit permission to publish them.

## Pull Requests

Before opening a pull request:

- Confirm the change is focused on one clear goal.
- Add or update tests, benchmark cases, or examples when behavior changes.
- Document known limitations and tradeoffs.
- Check generated or agent-authored code carefully before submitting it.

Maintainers may ask for narrower scope, more evidence, or clearer reproduction steps before merging.
